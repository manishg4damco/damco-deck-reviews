01

Project Overview

What is being built

JTR Aerospace Private Limited operates The Aviator School, an aviation pilot exam preparation academy in India. They are rebuilding their LMS from scratch to replace an existing OutSystems application. The new system covers DGCA certification exam preparation across SPL, PPL, CPL, ATPL, and Military tracks.

This is a full rebuild, not a migration. Damco is being engaged to design, build, and deliver the entire application.

Two phases, three portals

Phase 2FTO Portal (Flight Training Organization)React web · Multi-tenant with per-FTO subdomain

Phase 1 has two sub-phases: Phase 1A (Admin portal + Flutter mobile) and Phase 1B (Student web portal).

Key application characteristics
  • Offline-first mobile: All study content downloaded to device. Audio up to 25 MB/chapter, PDFs up to 50 MB. Background sync every 2 minutes. 2-day offline window.
  • Email: 40 templates exclusively via Microsoft 365 SMTP. Client owns M365 licence. Zero new email cost.
  • SMS OTP: Twilio, Indian numbers only. DLT registration with TRAI is a hard prerequisite (see Section 8).
  • Video: YouTube private embeds (client constraint C-01 in SRS). No self-hosted video.
  • Payments: Razorpay across all platforms. No Apple IAP or Google Play Billing.
  • Push notifications: FCM (Android) and APNs (iOS). Routed via AWS SNS.
  • Multi-tenancy Phase 2: Row-Level Security in PostgreSQL. Per-FTO subdomain via Route 53 wildcard.
  • Crash reporting & analytics: Firebase Crashlytics + Firebase Analytics. Both free.

ⓘ
GCP note The GCP figure above is based on the original comparison. After proper apples-to-apples spec matching (Cloud Run Tier 2 premium in asia-south1, Cloud SQL HA equivalent to RDS Multi-AZ), AWS is 10–20% cheaper at current scale. GCP's Tier 2 compute premium and Cloud SQL HA pricing offset its Redis and CI/CD savings. AWS is also confirmed as the correct choice for the 3-year horizon due to Cloud SQL being materially more expensive at larger DB instance classes.
Rationale: AWS over Azure
  • Azure Database for PostgreSQL Flexible Server (HA) is significantly more expensive than RDS Multi-AZ at equivalent spec (₹11,400/month vs ₹9,975/month).
  • Azure Front Door WAF costs ₹3,610/month vs AWS WAF at ₹950/month for equivalent protection.
  • Azure Container Apps has a minimum billing floor that penalises low-traffic Dev and UAT environments.
Rationale: AWS over GCP
  • Cloud Run (GCP) in asia-south1 is Tier 2 pricing — approximately 35% more expensive than US regions. With min-instances=2 (required for exam app availability), Cloud Run is more expensive than ECS Fargate.
  • AWS CloudFront has 33 India edge locations vs approximately 5–7 for GCP Cloud CDN — directly impacts audio file delivery speed (25 MB/chapter) across India.
  • AWS has the largest community and documentation base for the NestJS + PostgreSQL + Prisma stack, reducing operational risk for the build team.
Why ap-south-1 specifically

CloudFront has edge locations in Mumbai, Delhi, Chennai, Hyderabad, Bengaluru, and Kolkata. All students are in India. Audio file delivery (up to 25 MB/chapter) and PDF delivery benefit directly from pan-India CDN coverage with no additional configuration.

4.3 Data Layer
PostgreSQL — RDS Multi-AZ

Primary relational database for all application data. Multi-AZ in Production provides automatic failover in 60–120 seconds with zero data loss. Automated daily backups: 7 days (Production), 3 days (Dev/UAT). Point-in-time recovery on Production.

▲
Revalidated: Prod Year 1 upgraded to db.t3.medium db.t3.small has a 2 GB buffer pool and burstable CPU that depletes under sustained exam-season load. With 1K registered users and peak concurrent exam sessions (100–300 students), CPU credits exhaust and performance throttles. db.t3.medium (4 GB RAM, higher baseline CPU) prevents this. Upgrade path to t3.large is non-disruptive (brief maintenance window, typically under 5 minutes).

Redis — ElastiCache

Serves four functions simultaneously: session token storage, question bank cache (prevents repeat DB queries), rate-limiting counters (OTP abuse prevention), and BullMQ job queue (async email, push, invoice, badge processing).

⚠
Single node — accepted risk (no HA) Redis data is cache-only. A Redis failure means students re-authenticate; no business data is lost. Mandatory design guardrail: Exam answer saves and timer state must be written synchronously to PostgreSQL on every answer event. Redis holds a read cache only. BullMQ is reserved for non-critical async jobs (email, push notifications, invoice PDF, badge calculations). HA ElastiCache cluster adds ~₹4,750/month — not justified at startup scale.
